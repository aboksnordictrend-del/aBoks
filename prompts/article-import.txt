Каждый раз после этого сообщения я буду вставлять полный HTML-код одной готовой статьи.

Твоя задача — автоматически интегрировать её в проект aBoks.

## Что нужно сделать

1. Сам определить:

* заголовок статьи (из `<title>` или `<h1>`);
* meta description;
* slug (SEO-friendly, из заголовка).

2. Создать страницу статьи:

```
src/app/(frontend)/inspirasjon/[slug]/page.tsx
```

где `[slug]` соответствует названию статьи.

3. Не создавать отдельный `.html` файл.

4. Не использовать:

* `<!DOCTYPE>`
* `<html>`
* `<head>`
* `<body>`
* встроенный `<style>`

5. Преобразовать HTML в корректный React/Next.js TSX.

6. Использовать существующий layout сайта:

* Header
* Footer
* Breadcrumbs
* Typography
* Container
* Цвета
* Компоненты

Страница должна выглядеть как полноценная часть сайта aBoks.

## Контент

Полностью сохранить:

* H1
* H2
* H3
* абзацы
* списки
* таблицы
* blockquote
* FAQ
* callout-блоки
* внутренние ссылки
* внешние ссылки

При необходимости заменить обычные `<a>` на `next/link` для внутренних ссылок.

## Metadata

Автоматически создать metadata, используя информацию из HTML и SEO-пакета:

* title
* description
* canonical
* OpenGraph (og:title, og:description, og:image, og:image:width/height/type, og:url, og:type, og:site_name)
* Twitter Card (summary_large_image)

Обязательно:

1. Добавить статью и её главное изображение (с полями `imageWidth`/`imageHeight` — реальные пиксельные
   размеры картинки, минимум 1200×630) в `src/app/(frontend)/inspirasjon/_data.ts`.
2. В `page.tsx` статьи НЕ писать `openGraph`/`twitter` вручную. Вместо этого импортировать
   `buildArticleMetadata` из `../_seo` и вызвать его со slug статьи:

   ```ts
   import { buildArticleMetadata } from '../_seo'

   export const metadata: Metadata = buildArticleMetadata({
     slug: 'slug-statji',
     title: '...',
     description: '...',
   })
   ```

   `buildArticleMetadata` сам подставит og:image (из `_data.ts`, либо дефолтное изображение сайта,
   если картинки нет или она меньше 1200×630), абсолютные URL, og:type=article, og:site_name=aBoks
   и twitter:card=summary_large_image. Это гарантирует, что каждая новая статья автоматически получает
   корректные Open Graph и Twitter Card теги без ручной работы.

## Карточка статьи

После создания статьи она должна автоматически появиться в общем списке статей на странице **/inspirasjon**.

Карточка должна содержать:

- изображение (если есть);
- категорию;
- заголовок;
- краткое описание;
- дату публикации;
- slug.

Карточка должна вести на соответствующую страницу статьи.

Важно:
Не добавляй карточку вручную в код страницы.
Она должна автоматически отображаться через существующий список статей, который получает данные из
`src/app/(frontend)/inspirasjon/_data.ts` (статьи не хранятся в Payload CMS — добавь новую запись в этот
файл, и карточка появится сама).

Пагинация уже настроена по 9 статей на страницу. Новая статья должна автоматически появляться на первой
странице списка (если статьи отсортированы по дате публикации от новых к старым). При этом более старые
статьи должны автоматически смещаться на следующие страницы пагинации. Никаких дополнительных изменений
в пагинации или ручного распределения статей выполнять не нужно.

## Важно

* Не изменять существующий дизайн сайта.
* Не изменять Payload CMS.
* Не создавать новые коллекции.
* Не делать миграции.
* Не изменять структуру проекта.
* Использовать существующие компоненты и стили проекта везде, где это возможно.
* Если в HTML есть собственные стили, адаптировать их под дизайн aBoks, а не копировать встроенный CSS.

## SEO

Вместе с HTML статьи я также буду передавать готовый SEO-пакет.

Используй его без изменений.

SEO-пакет может содержать:

* Focus keyword
* Meta Title
* Meta Description
* SEO Description
* URL slug
* Suggested category
* Suggested tags
* Excerpt
* Hero image alt text

Используй эти данные для:

* metadata (title, description, Open Graph, canonical);
* карточки статьи на странице `/inspirasjon`;
* категории;
* slug;
* краткого описания статьи;
* alt-текста главного изображения (если оно используется).

Не генерируй новые SEO-данные самостоятельно, если они уже предоставлены. Всегда используй переданный SEO-пакет как основной источник информации.
